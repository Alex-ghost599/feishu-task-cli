# Security Policy

Do not report credentials, tokens, personal identifiers, or tenant data in a public Issue.

Use GitHub private vulnerability reporting for suspected vulnerabilities. Include the affected
version, a minimal synthetic reproduction, impact, and suggested mitigation. Do not test against
accounts, tenants, or data you do not own or have permission to use.

Security fixes follow the normal Issue/branch/PR workflow using a private advisory when public
disclosure would create risk.
